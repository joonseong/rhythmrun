# RhythmRun — 기획자 하네스

> 이 디렉토리는 제품 기획 문서 작업용입니다.
> Claude는 기획자 관점에서 제품 의사결정, 지표 설계, 범위 조율을 도와줍니다.

---

## 내 역할 컨텍스트

- 제품 비전과 MVP 범위를 정의하고 지킨다
- 개발팀·디자인팀에 "왜(Why)"를 전달한다
- 성공 지표를 설계하고 출시 후 검증한다

---

## 제품 비전

> "달릴 때마다 게임이 된다"

| 항목 | 내용 |
|------|------|
| 타겟 | 20–35세, 러닝에 재미·동기부여가 필요한 러너 |
| 핵심 차별점 | 가속도계 BPM → 실시간 오디오 리듬 피드백 |
| MVP 목표 | 재방문율 40% 이상 (7일 이내 2회 이상 세션 완료) |

---

## 성공 지표

| 지표 | 정의 | 목표 |
|------|------|------|
| 재방문율 (Primary) | 첫 세션 완료 후 7일 이내 2회 이상 세션 완료한 비율 | ≥ 40% |
| 리듬 피드백 활성화율 | 오디오 피드백 켠 채로 세션 완료한 비율 | ≥ 60% |
| 평균 세션 시간 | 완료된 세션의 평균 지속 시간 | ≥ 15분 |

**측정 기간**: 출시 후 4주차 코호트 기준

### 판단 기준

| 결과 | 액션 |
|------|------|
| 재방문율 ≥ 40% | V2 기능 개발 진행 |
| 재방문율 < 40% | 핵심 루프(리듬 피드백 → 재미 → 재방문) 재검토 |

---

## MVP 범위

### IN (Priority 1 — 반드시 출시)

- 가속도계 걸음 감지 + 실시간 BPM 계산
- BPM 연동 오디오 리듬 피드백 (비트가이드 / 자유런)
- 기본 트래킹: 시간, 거리, 페이스
- 결과 화면: 리듬 점수, 등급, BPM 그래프
- 소셜 공유: OS Share Sheet 이미지 카드
- 사용자 계정: 이메일 + Google + Apple 로그인

### OUT (V2 이후 — 협의 없이 추가 금지)

- GPS 트래킹, 코스 추천
- 친구 팔로우 / 소셜 피드
- 챌린지 시스템
- 음악 커스터마이징
- 심박수 연동 칼로리

## 화면 구조 (7개)

```
[스플래시] → [온보딩] → [로그인/회원가입] → [홈]
[홈] ↔ [러닝 준비] → [리듬 러닝 세션] → [결과]
[홈] ↔ [프로필/목표설정]
```

### 화면별 핵심 역할

| 화면 | 기획 목적 |
|------|----------|
| 홈 | 재방문 동기 부여 (기록, 점수, CTA) |
| 러닝 준비 | 세션 진입 마찰 최소화 |
| 리듬 러닝 세션 | 핵심 가치 경험 (BPM 피드백, 콤보) |
| 결과 | 성취감 + 공유 → 재방문 트리거 |
